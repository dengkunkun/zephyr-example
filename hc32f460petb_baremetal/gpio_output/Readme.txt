平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
使用步骤
================================================================================
1）打开工程并重新编译；
2）启动IDE的下载和调试功能，全速运行；
3) 观察LED状态：板载的4个LED灯循环的依次点亮和熄灭。

================================================================================
注意
================================================================================
1）当前目录下新增了一个独立的 GCC Makefile，可直接在命令行编译，不依赖 Eclipse/Embedded IDE：
    cd GCC
    make clean
    make -j2

    编译产物位于 GCC/build/ 目录，包括 gpio_output.elf、gpio_output.hex、gpio_output.bin 和 gpio_output.lst。

    若已安装 pyOCD，可继续直接执行：
    make flash
    make erase
    make gdbserver

    上述命令默认目标为 hc32f460xe。若需要指定探针，可追加：
    make flash PROBE=<probe-uid>

2）本工程的 LED 逻辑已按双灯板修改为：
    PD10 -> LED3
    PE15 -> LED4

    两个 LED 按“交替点亮”方式闪烁，且按实际硬件连接使用有源低输出。

3）经验证，pyOCD 可识别板载的“Open Cherry USB CMSIS-DAP”探针，并已成功完成一次实际下载。

4）经验证，标准版 OpenOCD 0.12 不包含 HC32F460 的片上 Flash 驱动，因此不能仅靠 stock OpenOCD 完成烧写。
    当前推荐使用 pyOCD（目标名 hc32f460xe）进行下载和调试；若后续拿到厂商定制的 OpenOCD target/flash 脚本，再补 OpenOCD 方案。
